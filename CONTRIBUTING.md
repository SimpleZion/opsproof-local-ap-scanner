# Contributing

Contributions are welcome when they improve local privacy, field mapping, explainability, or AP review usability.

Please do not submit private finance data, real vendor records, bank data, tax data, payroll data, customer data, or payment-card data.

Good contribution types:

- New demo-data fixtures.
- Better English or Chinese AP field aliases.
- UI copy that lowers non-technical buyer friction.
- Accessibility fixes.
- Deterministic duplicate-payment rule improvements.
- Tests for field mapping, report output, or vendor normalization.

Before opening a pull request, run:

```powershell
npm test
```
